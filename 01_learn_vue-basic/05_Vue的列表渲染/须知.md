#### v-for中的key
 - 便于vue对虚拟Dom中VNode的diff算法优化
 - 在vue的template中写的标签如
  - <div class="title" style="font-size: 30px; color: red;">哈哈哈</div>

    会被vue解析成一个javaScript对象

    const vnode = {
      type: "div",
      props: {
        class: "title",
        style: {
          "font-size": "30px",
          color: "red"
        }
      }
      children: "哈哈哈"
    }

     转化过程
   - template  -> VNode  ->  真实DOM
    - 原因: 
     - 1.方便跨平台(渲染在浏览器, 移动端, 桌面端的控件, VR设备)
     - 2.diff算法
